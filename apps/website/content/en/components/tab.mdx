<Section>
  <Section.Title>Overview</Section.Title>
  <Section.Text>Tab is a compound component that provides a keyboard-accessible tab interface following the WAI-ARIA pattern. Content panels are linked to their corresponding triggers via a shared `value` prop. It supports both controlled (<code>value</code> + <code>onValueChange</code>) and uncontrolled (<code>defaultValue</code>) usage.</Section.Text>
  <Section.UnorderedList>
    <Section.ListItem>Use Tab for switching between related content views within the same page context — for example, different settings categories or different data views.</Section.ListItem>
    <Section.ListItem>For step-by-step workflows, use a stepper pattern instead.</Section.ListItem>
    <Section.ListItem>Avoid nesting tabs within tabs.</Section.ListItem>
  </Section.UnorderedList>
</Section>

<Section>
  <Section.Title>Accessibility</Section.Title>
  <Section.UnorderedList>
    <Section.ListItem>Follows the WAI-ARIA Tabs pattern with <code>role="tablist"</code>, <code>role="tab"</code>, and <code>role="tabpanel"</code>.</Section.ListItem>
    <Section.ListItem><code>Arrow Left</code> / <code>Arrow Right</code> navigates between tab triggers.</Section.ListItem>
    <Section.ListItem><code>Home</code> moves focus to the first tab. <code>End</code> moves to the last tab.</Section.ListItem>
    <Section.ListItem>Tab panels are linked to their triggers via <code>aria-labelledby</code>, providing context for screen readers.</Section.ListItem>
    <Section.ListItem>Only the active tab panel is rendered in the accessibility tree; inactive panels are hidden.</Section.ListItem>
  </Section.UnorderedList>
</Section>

<Section>
  <Section.Title>Import</Section.Title>
