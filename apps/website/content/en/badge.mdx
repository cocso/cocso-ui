<Section>
  <Section.Title>Overview</Section.Title>
  <Section.Text>Badge is a small inline element that communicates status, category, or numeric quantity. It supports semantic color variants, three sizes, and three shape options, covering the most common labeling patterns.</Section.Text>
  <Section.UnorderedList>
    <Section.ListItem>Use <code>primary</code> and <code>secondary</code> for neutral category labels or tags.</Section.ListItem>
    <Section.ListItem>Semantic variants (<code>success</code>, <code>error</code>, <code>warning</code>, <code>info</code>) are ideal for status indicators.</Section.ListItem>
    <Section.ListItem>The <code>outline</code> variant works well for subtle, low-emphasis labels.</Section.ListItem>
    <Section.ListItem>Badge is a non-interactive display element — for clickable labels, use Button with an appropriate variant instead.</Section.ListItem>
  </Section.UnorderedList>
</Section>

<Section>
  <Section.Title>Accessibility</Section.Title>
  <Section.UnorderedList>
    <Section.ListItem>Badge renders as a <code>&lt;div&gt;</code> with no interactive role. Screen readers read the text content naturally.</Section.ListItem>
    <Section.ListItem>When the badge content updates dynamically (e.g., a notification count), consider adding <code>role="status"</code> to announce changes to assistive technologies.</Section.ListItem>
    <Section.ListItem>Ensure sufficient color contrast between the badge text and background, especially for the <code>outline</code> variant.</Section.ListItem>
  </Section.UnorderedList>
</Section>
