---
title: Popover
description: A floating panel anchored to a trigger element, built on Base UI's Popover primitive.
---

<Section>
  <Section.Title>Overview</Section.Title>
  <Section.Text>
    Popover is a compound component that operates in a non-modal manner, unlike Dialog, and does not block interaction with the rest of the page. Use it for supplementary information, inline forms, or rich tooltips.
  </Section.Text>
  <Section.UnorderedList>
    <Section.ListItem>Composed with a compound API: <code>Popover</code>, <code>Popover.Trigger</code>, <code>Popover.Content</code>, <code>Popover.Arrow</code></Section.ListItem>
    <Section.ListItem>Operates non-modally; clicking outside closes the popover without blocking interaction.</Section.ListItem>
    <Section.ListItem>Supports <code>side</code> and <code>align</code> positioning props.</Section.ListItem>
    <Section.ListItem><code>Popover.Arrow</code> renders an arrow caret pointing toward the trigger.</Section.ListItem>
  </Section.UnorderedList>
  <Section.UnorderedList>
    <Section.ListItem>Use Popover for supplementary information, inline forms, or rich interactive content anchored to a trigger.</Section.ListItem>
    <Section.ListItem>For short text-only hints, use Tooltip instead (lighter weight, hover-only).</Section.ListItem>
    <Section.ListItem>For content that demands user attention, use Dialog (modal, focus-trapping).</Section.ListItem>
  </Section.UnorderedList>
</Section>

<Section>
  <Section.Title>Accessibility</Section.Title>
  <Section.UnorderedList>
    <Section.ListItem>Non-modal by default — does not trap focus or block interaction with the rest of the page.</Section.ListItem>
    <Section.ListItem>Clicking outside or pressing <code>Escape</code> closes the popover.</Section.ListItem>
    <Section.ListItem>The trigger element receives <code>aria-expanded</code> to communicate the popover's open state.</Section.ListItem>
    <Section.ListItem>Focus management follows natural tab order within the popover content. Use with interactive elements like forms and buttons.</Section.ListItem>
  </Section.UnorderedList>
</Section>

<Section>
  <Section.Title>Import</Section.Title>
  ```tsx
  import { Popover } from "@cocso-ui/react";
  ```
</Section>

<Section>
  <Section.Title>Default</Section.Title>
  <Section.Text>A popover with a title, description text, and an arrow indicator.</Section.Text>
  <ComponentExample name="popover-default">
```tsx
"use client";

import { Button, Popover, Typography } from "@cocso-ui/react";

export default function PopoverDefault() {
  return (
    <div className="p-4">
      <Popover>
        <Popover.Trigger render={<Button variant="secondary" />}>Open Popover</Popover.Trigger>
        <Popover.Content>
          <div className="flex flex-col gap-2 p-3">
            <Typography size={14} weight="semibold">Popover Title</Typography>
            <Typography size={13}>This is the popover content. It can contain any elements.</Typography>
          </div>
          <Popover.Arrow />
        </Popover.Content>
      </Popover>
    </div>
  );
}
```
  </ComponentExample>
</Section>

<Section>
  <Section.Title>Positions</Section.Title>
  <Section.Text>Use the <code>side</code> prop on <code>Popover.Content</code> to control which side of the trigger the popover appears on.</Section.Text>
  <ComponentExample name="popover-positions">
```tsx
"use client";

import { Button, Popover, Typography } from "@cocso-ui/react";

export default function PopoverPositions() {
  return (
    <div className="flex flex-wrap gap-3 p-4">
      {(["top", "bottom", "left", "right"] as const).map((side) => (
        <Popover key={side}>
          <Popover.Trigger render={<Button variant="outline" />}>{side}</Popover.Trigger>
          <Popover.Content side={side}>
            <div className="p-3">
              <Typography size={13}>Popover on {side}</Typography>
            </div>
            <Popover.Arrow />
          </Popover.Content>
        </Popover>
      ))}
    </div>
  );
}
```
  </ComponentExample>
</Section>
